Declaration

type TVec4 = packed object(TObject)

Description

TVec4 is a basic vector object wich can be modified with operators and functions and which provides some methods itself. There are provided different ways to access the fields of the vector.

  1. Using Coord

  2. Using x,y,z,w r,g,b,a s,t,p,q properties

  3. Using Swizzling

1. Access the fields with the array Coord

Coord is the default property so using v[ 0 ] is the same as v.Coord[ 0 ] .

2. Alternatively you can use it's properties X, Y, Z, W .

However you can also write R, G, B, A if you use the vector for color values for example. Another alias is S, T, P, Q which is for texture coordinates. So all these code lines are equivalent: 

v.Coords[ 0 ]:= 0.0;
v[ 0 ]:= 0.0;
v.x:= 0.0;
v.r:= 0.0;
v.s:= 0.0;

These different type of access are only for convenience but point to one and the same field. Except for the Coord alias everything works just like in glsl .

Depending on the vectors type some fields may not be availabe.
For TVec2 there is only X (R, S) and Y (G, T) defined.
For TVec3 there is only X (R, S), Y (G, T) and Z (B, P) defined.
For TVec4 there is X (R, S), Y (G, T), Z (B, P) and W (A, Q) defined.


3. Additionally math3d supports swizzling in glsl-style .

This means you can write: 

v2:= v1.xxy; // equivalent to vec3( v1.x, v1.x, v1.y )

Unlike glsl this is not yet supported: 

var
  v1: TVec2;
  v2: TVec3;
begin
  v2:= v1.xxx; //Returns an error because v1 only has two members and cannot return a TVec3
end;

This is due to an internal problem of how the math3d unit is written. In Pascal you can only forward declare classes and procedures but not objects. So it is not possible for TVec2 to return TVec3 because it's still undefined at that stage. There is no solution for that for now i fear.

For more information on swizzling see this doc: https://www.opengl.org/wiki/Data_Type_%28GLSL%29#Swizzling

Description

Methods

function GetDist: Float;

Calculate the length of the current vector.

Returns

The result of the function.

function Normalize: TVec4;

Normalizes the current vector.

For returning a normalized vector without changing the current one use Vecnormalize

Returns

The result of the function will be returned additionally.

function CrossProduct( v: TVec4 ): TVec4;

Calculate the cross product of the current vector and v, wheras the current vector is the first and v is the second.

Returns

The result of the function.

function DotProduct( v: TVec4 ): Float;

Calculate the dot product of the current vector and v, wheras the current vector is the first and v is the second.

Returns

The result of the function

function GetBiggestComp: Integer;

Determines the biggest component (X, Y, Z, W).

Returns

The the index of the biggest component which can be used with Coord 

s:= v.Coord[ v.GetBiggestComp ];

function Invert: TVec4;

Negates the current vector without altering it.

Returns

The result is only returned without altering the original vector. This is exivalent to 

v2:= -v

function Ptr: PFloat;

Function to get access to the first element (X) of the vector.

Returns

The result is a pointer to the data of a vector. It can be used when passing to opengl for example. This is equivalent to 

ptr:= @v.Coord[ 0 ];

function Compare( v: TVec4 ): Boolean;

Compares two vectors and returns if there field values are equal.

Returns

The result is a boolean and is True if the vectors are equal.

function ToString: String;

This will generate a string with the vector's values enclosed in <>. It can be used for debugging or printing the values of a vector on screen.

Returns

The string is return in the function's result
